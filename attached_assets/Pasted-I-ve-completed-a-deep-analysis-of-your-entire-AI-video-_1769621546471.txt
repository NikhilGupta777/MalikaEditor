I've completed a deep analysis of your entire AI video pipeline. Here's what I found:
Current State: Video Analysis IS Being Generated, But NOT Fully Used
What's Working Well ✅

    Basic video analysis (scenes, keyMoments, emotionFlow) is used in edit planning prompts
    VideoContext (genre, tone) is used for AI image generation styling
    Post-render self-review - AI watches final video, scores it, can trigger corrections and re-render (max 2 iterations)
    Semantic analysis (topics, hooks, B-roll windows) flows to edit planning and media selection

Critical Gaps Found ❌

1. Enhanced Analysis Data is GENERATED but NEVER USED
The full video watching system creates rich data that's completely ignored:

    motionAnalysis (action sequences, motion intensity) - Not used for deciding video vs image for B-roll
    transitionAnalysis (detected transitions, suggested points) - Not used for transition placement
    pacingAnalysis (pacing variations, adjustment suggestions) - Not used for B-roll timing/duration
    audioVisualSync (sync quality, out-of-sync moments) - Not used for cut adjustments

2. Media Selection Relies on Keyword Heuristics Only
Currently uses basic word matching ("motion", "running", "walking") instead of actual motionAnalysis data from video watching.

3. B-Roll Duration is Fixed, Not Intelligent
Edit plan specifies B-roll start/end, but doesn't use pacingAnalysis to adjust duration based on content pacing.

4. enhancedAnalysis Not Persisted
The rich motion/transition/pacing data is generated, broadcast once via SSE, then lost - not saved to database.

    Store enhancedAnalysis in project database
    Pass motionAnalysis, transitionAnalysis, pacingAnalysis to edit planning prompts

    Use pacing suggestions for B-roll timing

Phase 2: Intelligent Media Selection

    Use motionAnalysis to decide video vs image (high motion = prefer video)

    Include enhancedAnalysis data in self-review context
