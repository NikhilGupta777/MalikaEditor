Complete Bug & Issue Report
CRITICAL BUGS
1. AI Images Generated but Never Used (Waste of API Credits)

Your logs show: WARNING: 7 AI images were generated but NONE were selected
Root cause: The media selector has a strong bias toward video over static images. When it sees "motion" content, it always picks stock videos over AI-generated images. So the system spends API credits generating 7 images with Gemini, then throws them all away.
Files: server/services/ai/mediaSelector.ts (line 463), backgroundProcessor.ts
2. Caption Spelling Fixes Are a No-Op

The self-review correctly detects the "Khan Pralay" vs "Khand Pralay" inconsistency and marks it AutoFixable: true
But the fix does nothing. The adjust_caption handler just logs "Caption adjustment queued (requires re-render)" — it never actually modifies the transcript text
If it triggers a re-render, it re-renders with the same misspelled transcript, wasting time and compute
File: server/services/ai/postRenderReview.ts (line 614-617)
3. Pre-render Review JSON Parsing Failure

Your logs show: Attempt 1: Could not extract JSON from response — the AI wraps its response in markdown code blocks (```json)
The parser eventually recovers, but on the first attempt it fails because it can't strip the markdown fence properly
When parsing fully fails, it defaults to auto-approve with 75% confidence, which means bad edits can slip through
File: server/services/ai/preRenderReview.ts
VIDEO PLAYER BUGS
4. No Keyboard Shortcuts at All

YouTube has: Space (play/pause), Left/Right arrows (seek), M (mute), F (fullscreen), 0-9 (seek to %), J/K/L (seek/play)
Your player has zero keyboard support. No keydown listener exists anywhere in VideoPreview.tsx
5. No Playback Speed Control

No 0.5x/1x/1.5x/2x speed selector. The playbackRate property is never set on the video element
YouTube's speed control is one of the most-used features
6. Video Loading Hack

Uses a setTimeout(500ms) to force setIsLoaded(true) — this is unreliable and can show controls before the video is actually playable

But the self-review DID find an issue (the Khan/Khand Pralay spelling). The AI chat doesn't have access to the self-review results in its context properly, or the self-review data wasn't saved to the DB before the chat reads it
