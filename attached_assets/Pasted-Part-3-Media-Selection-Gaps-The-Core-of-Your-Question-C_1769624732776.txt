Part 3: Media Selection Gaps (The Core of Your Question)
Current Media Decision Flow:

B-roll window identified →
  Fetch stock media (Pexels/Freepik) →
  Generate AI images →
  AI Vision analyzes thumbnails →
  AI prompt asks "which is best?" →
  Selection made

What's Missing:

    No Scene-Aware Selection: The AI doesn't know what the VIDEO looks like at that moment - it only knows the transcript context. It should consider:
        What visuals are currently on screen?
        Would B-roll complement or clash with current scene?
        Is the speaker gesturing (don't cover important gestures)?

    No Emotion-Matched Selection: If the speaker is expressing excitement, choose energetic/dynamic media. If somber, choose calming imagery.

    No Duration-Aware Selection: Stock videos have durations. Currently selects based on content match but not optimal duration match.

    No "Is B-roll Even Needed Here?" Decision: AI should decide IF B-roll is needed, not just WHICH B-roll. Some moments are better without overlay.

    No Visual Continuity: AI doesn't ensure B-roll clips flow visually (color palette, brightness, style).

Proposed Smart Media Selection System:

interface IntelligentMediaDecision {
  windowId: string;
  shouldUseBroll: boolean;  // NEW: AI decides IF to use B-roll
  reasoning: string;
  if shouldUseBroll:
    preferredType: "video" | "image" | "ai_generated";
    typeReasoning: string;  // "Video because high motion scene"
    selectedMedia: MediaCandidate;
    selectionReasoning: string;
    visualContinuityScore: number;  // How well it matches surrounding content
    emotionalMatch: number;  // How well it matches emotional tone
}

Part 4: Learning System Improvements
Current Limitations:

    Only stores pattern type (cut/transition/broll) with basic context
    Retrieval uses simple genre/tone matching
    Doesn't learn from user rejections (only approvals)
    Doesn't learn from self-review failures
    No reinforcement from actual video performance

Proposed Improvements:

    Learn from Rejections: When user un-approves a cut, learn "don't cut here in similar contexts"

    Learn from Self-Review Issues: If self-review finds "jarring cut at 15s", feed that back to cut decision model

    Cross-Project Learning: "In tech tutorial videos, B-roll with code screenshots gets 90% approval rate"

    Time-Based Learning: "Users prefer shorter B-roll in first 30 seconds, longer in middle"

    Style Clusters: Group successful edits by visual style, not just genre/tone
